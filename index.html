Spreadsheet Setup

  • Format: Use CSV or Excel (.xlsx). The first row must be column headers — these become your card field names.
  • One row = one card. Every row below the header will generate a card in the deck.
  • Art column: Name a column art, image, or artwork and fill each cell with the filename of that card's art image (e.g. dragon.png). Deck Architect will match it to your uploaded art automatically.
  • Icon tokens: Type {tokenname} anywhere in a cell to embed an inline icon. Upload your icons and assign token names in the Icon Library first.
  • Card types / profiles: Add a column like Type (e.g. Spell, Creature, Item) to enable Profile Logic — different card types can then have their own frames and layouts automatically.
  • Amount column: Add a column named amount to pre-set how many copies of each card are included in a full deck export. This is display-only — it never renders on the card face.
  • Rich text (Excel only): Bold, italic, and underline formatting applied inside individual Excel cells is preserved when the file is uploaded.
  • Column names matter: Keep them short and consistent — they appear as field labels throughout the tool and are used as keys for art and icon matching.

Loading Your Data

  • Upload a CSV or Excel (.xlsx) file in the Database section. Each row = one card; column headers = field names.
  • Use PREV / NEXT or the dropdown at the bottom of the sidebar to browse cards.
  • Add a column named art or image to auto-link artwork by filename.
  • Excel bold/italic/underline is preserved automatically.
  • You can Edit Card data directly inside the tool — click the Edit Card button in the sidebar to update any field value for the current card without touching your spreadsheet.

Manual Cards

  • No spreadsheet? No problem. Click + Add Card in the sidebar to create cards from scratch.
  • A modal will prompt you to fill in any available fields. Leave fields blank and edit them later using the Edit Card button.
  • Manual cards can be combined with imported spreadsheet data — you can add blank cards to an existing deck at any point.
  • Set a card to 0 copies to use it as a draft or placeholder without including it in exports.
The first time you add a manual card with no spreadsheet loaded, a Name field is created automatically.

Frame Assets

  • Upload transparent PNG overlays (borders, card chrome) via the Frame Assets drop zone.
  • Click a thumbnail to apply it as the active frame. Hover + click ✕ to remove.
  • Each uploaded frame also has a ▣ Card Back button beneath it — click it to designate that image as the card back template. Click again to restore the default.
  • Frames can be assigned per profile — activate a profile first, then click a frame to set it for that card type only.

Art Assets

  • Upload card illustrations (PNG, JPG, WebP) via the Art Assets drop zone in the sidebar.
  • Once uploaded, click Browse & Assign Art to open the art library and click any image to assign it to the current card. Click again to unassign.
  • The currently assigned art shows as a small preview in the sidebar so you always know what's set for the current card.
  • Or use an art column in your spreadsheet — Deck Architect matches by filename or the name set in the library.
  • Each art entry has a rename field — give images friendly names for easier identification in the library.

Icon Library

  • Upload an icon image, give it a token name (e.g. fire), then type {fire} in any spreadsheet cell or inline editor to embed it inline.
  • Drag the ⊿ corner handle on an icon entry to adjust its display size.
  • Icons scale proportionally with font size and are baked into the exported PNG.

Edit Layout

  • Click Edit Layout in the header to enter layout mode. Click it again to exit.
  • Move mode (default) — click a field to select it (highlighted in orange), then drag it to reposition. Drag the orange corner handle to resize the field box.
  • Style mode — toggle the Style switch in the toolbar to access font family, font size, colour, bold, italic, underline, and text alignment controls.
  • Inline edit — double-click a field while in Style mode to edit its text directly on the card. Select text to get a bold/italic/underline popup.
  • Target dropdown: Global applies changes to all cards, Profile applies to this card type only, Card applies to this card only. The ↺ reset button next to the scope label clears any card- or profile-level override and reverts to global.
  • The coordinates display in the toolbar shows the selected field's current position and size in pixels for precise placement.
Changes are undoable — use Ctrl+Z / Ctrl+Y or the undo/redo buttons in the header.

Keyboard Shortcuts

  • Arrow keys — with a field selected in Edit Layout (Move mode), nudge it 1px per press in any direction.
  • Shift + Arrow keys — nudge the selected field 10px per press for faster repositioning.
  • Ctrl+Z — undo the last change.
  • Ctrl+Y (or Ctrl+Shift+Z) — redo.
  • Escape — close any open modal (Add Card, Edit Card).
Keyboard shortcuts are disabled while typing in an input field or the inline editor so they don't interfere with text entry.

Font Choices

In Style mode, select a field and use the Font dropdown in the toolbar to change its typeface. Fonts are grouped by theme and tagged by role — when you pick a font the toolbar shows a colour-coded hint: green = Body, purple = Title, amber = Flavor.

Body — readable at small sizes for rules and card descriptions. Title — decorative and bold, best for card names at larger sizes. Flavor — elegant and lightweight, ideal for italicised lore or quote text.

Edit Art

  • Click Edit Art in the header to enter art edit mode.
  • Use the Art Scale slider to zoom the artwork in or out (50–300%).
  • Click and drag directly on the card to pan the image into position.
  • Click ↺ Reset to return the art to 100% scale, centred.
Scale and pan position are saved per card and baked into the exported PNG. Art edits do not affect other cards.

Layer List

  • The Layer List in the sidebar shows all fields from your spreadsheet as toggleable rows.
  • Click the eye icon on any row to hide or show that field on the card.
  • Visibility respects the active Target scope — you can hide a field globally, for one profile type, or for a single card only.
  • Use this to suppress columns you don't want rendered (e.g. internal reference fields or the art column) without removing them from your data.

Profile Logic

  • Pick a column (e.g. Type) under Profile Logic in the sidebar — a profile is created for each unique value in that column.
  • Click a profile chip to activate it. The header shows the active profile context.
  • With a profile active, use Edit Layout with Target: Profile to set field positions, sizes, fonts, and visibility for just that card type.
  • You can assign a different frame to each profile — activate the profile first, then click the frame thumbnail.
Profile overrides sit between Global and Card scopes. Card-level changes always take highest priority.

Card Back

  • Every deck automatically includes a Card Back as its final entry — navigate to it using NEXT or the card selector dropdown.
  • When viewing the card back, a purple banner appears above the preview. Global layout rules do not apply here — it is a fully independent canvas.
  • To set a custom card back image, upload your art and assign it via Browse & Assign Art, just like any other card.
  • To use a frame template as the card back design, click the ▣ Card Back button beneath any frame thumbnail in Frame Assets. Click it again to restore the default.
  • Use the toggle on the card back banner to include or exclude the card back from Print Sheet and Full Deck exports. TTS export always includes it automatically.

Export Copies

  • The Export Copies control (the + / − buttons at the bottom of the sidebar) sets how many copies of the current card are included in a full deck export.
  • Set a card to 0 to exclude it from exports entirely — useful for placeholder or work-in-progress cards.
  • You can pre-set copy counts in your spreadsheet by adding a column named amount. Deck Architect reads this on load and reflects it in the control.
Export Copies only affects Export Full Deck, Print Sheet, and TTS outputs — it has no effect on Export Current.

Exporting

  • Export Current — downloads a full-resolution PNG of the card currently on screen.
  • Export Full Deck — renders every card (respecting Export Copies counts) and downloads them as a ZIP of individual PNGs.
  • Print Sheet – Letter — lays all cards out on landscape US Letter pages (3 columns × 3 rows per page) and downloads a .docx Word file, ready to print and cut. Each card is sized at 2.49" × 3.47" (standard poker card).
  • Export for TTS — see the TTS section below.
Don't close the tab while the export overlay is running — all rendering happens in your browser.

Tabletop Simulator (TTS)

  • Export for TTS — stitches all cards into 10×7 grid sheets (up to 69 cards per sheet) at print resolution and downloads them as PNGs. The card back is placed in slot 70 automatically. Multiple sheets are generated if your deck exceeds 69 cards.
  • Load TTS Sheets — import cards from existing TTS sprite sheets. Click the button in the sidebar, select one or more TTS sheet PNGs, and Deck Architect will slice each slot into individual art assets and add them to your deck automatically. Blank slots are detected and skipped.
TTS sheet images must be standard TTS format: 10 columns × 7 rows (plus the hidden card in slot 70). The importer checks pixel variance to skip empty slots.

Save & Load

  • Save Project — downloads a .deckproject.zip containing all your data, art assets, icon assets, frame templates, layout settings, profile overrides, card overrides, and copy counts.
  • Load Project — restores everything from a previously saved .deckproject.zip, including all images and configuration.
There is no auto-save. Save your project regularly — especially before exporting large decks.
