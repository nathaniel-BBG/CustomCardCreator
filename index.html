Spreadsheet Setup

  • Format: Use CSV or Excel (.xlsx). The first row must be column headers — these become your card field names.
  • One row = one card. Every row below the header will generate a card in the deck.
  • Art column: Name a column art, image, or artwork and fill each cell with the filename of that card's art image (e.g. dragon.png). Deck Architect will match it to your uploaded art automatically.
  • Icon tokens: Type {tokenname} anywhere in a cell to embed an inline icon. Upload your icons and assign token names in the Icon Library first.
  • Card types / profiles: Add a column like Type (e.g. Spell, Creature, Item) to enable Profile Logic — different card types can then have their own frames and layouts automatically.
  • Rich text (Excel only): Bold, italic, and underline formatting applied inside individual Excel cells is preserved when the file is uploaded.
  • Column names matter: Keep them short and consistent — they appear as field labels throughout the tool and are used as keys for art and icon matching.

Loading Your Data

  • Upload a CSV or Excel (.xlsx) file in the Database section. Each row = one card; column headers = field names.
  • Use PREV / NEXT or the dropdown at the bottom of the sidebar to browse cards.
  • Add a column named art or image to auto-link artwork by filename.
  • Excel bold/italic/underline is preserved automatically.

Frame Assets

  • Upload transparent PNG overlays (borders, card chrome) via the Frame Assets drop zone.
  • Click a thumbnail to apply it as the active frame. Hover + click ✕ to remove.

Art Assets

  • Upload card illustrations (PNG, JPG, WebP) via the Art Assets drop zone in the sidebar.
  • Once uploaded, click Browse & Assign Art to open the art library and click any image to assign it to the current card. Click again to unassign.
  • The currently assigned art shows as a small preview in the sidebar so you always know what's set for the current card.
  • Or use an art column in your spreadsheet — Deck Architect matches by filename or the name set in the library.

Icon Library

  • Upload an icon, give it a token name (e.g. fire), then type {fire} in any spreadsheet cell to embed it inline.
  • Drag the ⊿ handle on an icon entry to adjust its size.

Edit Layout

  • Move mode — drag a field to reposition it; drag the orange corner to resize.
  • Style mode — toggle the Style switch to change font family, font size, colour, bold, italic, and alignment.
  • Inline edit — double-click a field in Style mode to edit text directly. Select text for the bold/italic/underline popup.
  • Target dropdown: Global (all cards), Profile (this type), or Card (this card only).

Font Choices

In Style mode, select a field and use the Font dropdown in the toolbar to change its typeface. The table below summarises each option:

Font Option Best For "Vibe" Print Reliability
Atkinson Hyperlegible Accessibility / Default Functional & Clear Highest
Libre Baskerville Fantasy / Traditional Scholarly & Epic High
Archivo Narrow Complex Rules / Wordy Cards Industrial & Tight Medium-High
Inter Tight Modern / Sci-Fi Sleek & Techy High
Font choice respects the Target scope — you can set a different font per field, per profile, or per individual card.

Edit Art

  • Use the Art Scale slider to zoom the artwork (50–300%).
  • Click and drag on the card to pan the image into position.
  • Click ↺ Reset to return to 100% centred.
Scale and position are saved per card and baked into the exported PNG.

Profile Logic

  • Pick a column (e.g. Type) under Profile Logic — a profile is created for each unique value.
  • Click a profile to activate it, then use Edit Layout with Target: Profile to style just that card type.
  • You can assign a different frame to each profile.

Exporting

  • Export Current — downloads a PNG of the current card.
  • Export Full Deck — renders all cards and downloads them as a ZIP.
Don't close the tab while the export overlay is running.

Save & Load

  • Save Project — downloads a .deckproject.zip with all your data, images, and settings.
  • Load Project — restores everything from a saved file.
No auto-save — remember to save regularly.
